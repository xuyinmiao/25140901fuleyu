 

在本实验中，LLM 主要承担什么⻆⾊？它如何借助语义与常识，缓解纯符号执⾏在搜索空间上的困难？                                                                                                                                                                                                                                               

在本实验中，LLM 主要承担的是高层决策与工具编排角色。   它本身不直接完成二进制指令级分析，也不负责真正求解约束；这些工作由 angr和底层求解器完成。LLM 的作用是根据任务目标和上一轮 Observation，决定下一步该调用哪个工具，比如先  检查目标程序、再探索成功路径、最后从成功状态中求解输入并验证结果。   它缓解纯符号执行搜索空间困难的方式主要是利用语义信息和常识来指导搜索。比如程序里出现了：   Success!  Wrong password!  trapped  dead loop  check_password   LLM 可以理解这些字符串的含义，知道Success!通常代表目标路径，Wrong password!代表失败路径，trapped、dead loop代表应该避开的陷阱路径。于是它可以让 angr：   find: 输出包含 Success! 的路径  avoid: 输出包含 Wrong password! 或 trapped 的路径   这样 angr 不再盲目遍历所有路径，而是在 LLM 提供的语义方向下进行受控探索，减少进入错误分支、死循环分支和无关路径的机会。   简单说就是：   LLM 负责“看懂目标、决定方向”  angr 负责“严格执行路径探索和约束求解”   二者结合后，既利用了 LLM 的语义理解能力，也保留了符号执行在代码约束求解上的严密性。
